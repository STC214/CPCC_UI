============

本地漫画文件处理工具需求文档  
1. 程序概述    
开发一个整合cleaner.py、old.py和new.py功能的桌面应用程序  
提供两种文件处理流程  
具有现代化UI界面  
并打包为独立exe可执行文件      
3. 核心功能需求  
2.1 双流程处理机制  
新方法流程：执行顺序为 cleaner.py → new.py  
new.py中所有需要输入'y'确认的地方，自动默认输入'y'  
旧方法流程：执行顺序为 cleaner.py → old.py
UI上提供两个明确标识的按钮："新方法"和"旧方法"，用于触发对应流程  
2.2 实时进度显示  
显示实时文本日志，完整呈现命令行输出内容  
配备进度条，根据处理阶段动态更新  
进度信息需准确反映cleaner.py、old.py或new.py当前执行状态  
2.3 路径显示  
UI中包含单独一行显示"当前工作目录：[路径]"  
路径必须是exe文件所在目录，而非临时解压目录  
4. UI/UX设计规范  
3.1 视觉风格  
采用Darcula PyCharm Theme for VS Code主题配色方案：  
背景色：#2B2B2B  
主文字色：#A9B7C6  
次要文字/注释色：#808080  
选中/高亮色：#214283  
按钮默认色：#3C5A7C  
按钮悬停色：#4C6A8C  
按钮激活色：#214283  
边框色：#313335  
进度条填充色：#6897BB  
进度条背景色：#3C3F41  
标题栏必须遵循相同配色主题  
3.2 界面布局  
1. 顶部：应用标题和图标  
2. 第二行：显示当前工作目录（只读文本框或标签）  
3. 中部：两个主要功能按钮（"新方法"和"旧方法"），水平排列  
4. 底部区域：  
进度条（占满宽度）  
日志输出区域（可滚动文本框）  
4. 技术实现要求  
4.1 资源集成  
将cleaner.py、old.py、new.py全部打包到exe中  
程序不依赖外部python文件，完全自包含  
4.2 图标处理  
使用当前目录下的04.ico作为程序图标  
图标必须正确显示在：  
应用程序标题栏  
Windows任务栏  
exe文件自身  
4.3 工作目录处理  
程序必须识别并使用exe所在目录作为工作目录  
获取路径方式：os.path.dirname(sys.executable)（打包后）或os.path.dirname(os.path.abspath(__file__))（开发时）  
4.4 子进程交互  
通过subprocess模块执行内部脚本  
实时捕获并显示stdout/stderr  
对new.py的交互式提示自动提供确认（'y'）  
5. 打包规范  
生成单文件exe（减少依赖文件）  
正确处理资源文件嵌入  
兼容Windows 10/11主流版本  
体积优化，避免不必要的依赖  
6. 测试要求  
验证两种处理流程正常执行  
验证UI在处理过程中不冻结  
验证进度信息准确同步  
验证图标在各位置正确显示  
验证工作目录识别正确  
验证在无Python环境的纯净Windows系统上可正常运行  
7. 交付物  
完整的项目源代码  
配置完善的打包脚本  
最终生成的exe可执行文件  
简明使用说明（可选）
